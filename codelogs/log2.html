Code Log 2: CSS Measuring Units

Q: What is the difference between absolute and relative CSS length units?

CSS length units help define the size of different elements in your code. When you want to add an object to your code with a specific size, you will use length units to determine how big that object will be.

For example:

Absolute CSS length units are units that do not depend on anything else in your code to determine the size. Comparably, one foot is equivalent to twelve inches in the same way that one cm as a CSS length unit is equal to 37.8px and 25.2/64 in.

Relative CSS length units then are relative to another element of the code. If you want to change the size of something by comparing it to the size of something else, you might use relative length units. For font size, em determines font size relative to the font size of the original font-size element.

TLDR- semantic tags tell you what they will contain, non-semantic tags depend on the text within them to determine what they will contain.

Q: What are two (or more) benefits of using semantic tags?

One benefit is being more clear and concise. Using the same kind of tag to do multiple different things can make your code harder to understand and more difficult to search through for errors. This also applies with transferring your code to different browsers, between different coders, and within different contexts. Semantic tags are more versatile.

Another benefit is better accessibility. Content that cannot be accessed by everyone is content that will not be viewed by as many people.
